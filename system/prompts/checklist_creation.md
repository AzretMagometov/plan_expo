# Промпт для создания чеклистов

## Задача
Создать практичные чеклисты для отслеживания прогресса на всех трёх уровнях.

## Принципы создания чеклистов

1. **Визуальность** — легко видеть прогресс
2. **Конкретность** — чёткие действия, не размытые задачи
3. **Измеримость** — можно отметить выполнение
4. **Мотивация** — видеть прогресс вдохновляет
5. **Гибкость** — можно адаптировать под изменения

## Структура чеклистов

### Уровень 1: Стратегический чеклист (Identity Evidence List)

**Цель:** Отслеживать доказательства новой идентичности

**Формат:**
- Список маленьких действий, подтверждающих идентичность
- Отмечать каждое выполненное действие
- Праздновать каждые 10 пунктов

**Пример для "разработчик":**
```
┌─────────────────────────────────────────┐
│ Identity Evidence List                 │
│ Идентичность: Я — разработчик, который │
│ создаёт полезные продукты              │
├─────────────────────────────────────────┤
│ ☐ Написал код с комментариями          │
│ ☐ Исправил баг и написал тест          │
│ ☐ Поделился решением в сообществе      │
│ ☐ Сделал code review коллеге           │
│ ☐ Изучил новый паттерн                 │
│ ...                                     │
│ Празднование: каждые 10 пунктов! 🎉    │
└─────────────────────────────────────────┘
```

### Уровень 2: Тактический чеклист (OKR/SMART Tracker)

**Цель:** Отслеживать прогресс по Key Results или SMART-целям

**Формат для OKR:**
- Текущее значение / Целевое значение
- Процент выполнения
- Еженедельное обновление

**Пример:**
```
┌─────────────────────────────────────────┐
│ OKR Tracker (Q1 2024)                   │
├─────────────────────────────────────────┤
│ Objective: Стать экспертом в UX         │
│                                          │
│ KR1: Опубликовать 12 статей            │
│      Прогресс: 3/12 (25%) ✅            │
│      Последнее обновление: 15.01        │
│                                          │
│ KR2: Выступить на 3 конференциях        │
│      Прогресс: 0/3 (0%) ⏳              │
│      Заявки поданы: 2                   │
│                                          │
│ KR3: Набрать 5000 подписчиков          │
│      Прогресс: 1200/5000 (24%) ✅       │
│      Текущий рост: +50/неделю           │
│                                          │
│ Общий прогресс: 16%                     │
│ Цель: 70% к концу квартала              │
└─────────────────────────────────────────┘
```

**Формат для SMART:**
```
┌─────────────────────────────────────────┐
│ SMART Goal Tracker                      │
├─────────────────────────────────────────┤
│ Цель: Пройти курс Python до 1 июня     │
│                                          │
│ Прогресс: 45% (9 из 20 модулей)        │
│                                          │
│ Еженедельные цели:                      │
│ ☑ Неделя 1: Модули 1-3                 │
│ ☑ Неделя 2: Модули 4-6                 │
│ ☐ Неделя 3: Модули 7-9                 │
│ ☐ Неделя 4: Модули 10-12               │
│ ...                                     │
│                                          │
│ Следующий чекпоинт: 22.01               │
└─────────────────────────────────────────┘
```

### Уровень 3: Операционный чеклист (Daily Habits Tracker)

**Цель:** Отслеживать ежедневные действия и привычки

**Формат:**
- Еженедельная таблица
- Отмечать выполнение каждый день
- Видеть streak (серии дней)

**Пример:**
```
┌─────────────────────────────────────────┐
│ Daily Habits Tracker (Неделя 15-21 янв) │
├─────────────────────────────────────────┤
│ Привычка          │ Пн │ Вт │ Ср │ ... │
├───────────────────┼────┼────┼────┼─────┤
│ Утренний кодинг   │ ✅ │ ✅ │ ⏸  │ ... │
│ LeetCode задача   │ ✅ │ ✅ │ ✅ │ ... │
│ Статья по програ. │ ✅ │ ⏸  │ ✅ │ ... │
│ Code review       │ ⏸  │ ✅ │ ⏸  │ ... │
├───────────────────┴────┴────┴────┴─────┤
│ Streak: 5 дней подряд! 🔥              │
│ Выполнение: 75%                        │
└─────────────────────────────────────────┘
```

**Формат для If-Then планов:**
```
┌─────────────────────────────────────────┐
│ Implementation Intentions Tracker       │
├─────────────────────────────────────────┤
│ ЕСЛИ открыл ноутбук → ТО кодинг 2ч     │
│ Выполнено: 12 из 14 дней (86%) ✅      │
│                                          │
│ ЕСЛИ застрял >30мин → ТО вопрос в ком. │
│ Выполнено: 5 раз за неделю ✅           │
│                                          │
│ ЕСЛИ пятница → ТО code review          │
│ Выполнено: 3 из 3 недель ✅            │
└─────────────────────────────────────────┘
```

## Процесс создания чеклистов

### Шаг 1: Анализ целей

Проанализируй созданную трёхуровневую систему и определи:
- Какие действия нужно отслеживать на каждом уровне
- Как часто обновлять (ежедневно, еженедельно, ежемесячно)
- Какие метрики важны

### Шаг 2: Создание структуры

Для каждого уровня создай:
1. **Стратегический:** Identity Evidence List
2. **Тактический:** OKR/SMART Tracker с метриками
3. **Операционный:** Daily Habits Tracker

### Шаг 3: Определение частоты обновления

- **Стратегический:** еженедельно или при выполнении действия
- **Тактический:** еженедельно (обновление метрик)
- **Операционный:** ежедневно (отметка выполнения)

### Шаг 4: Добавление мотивационных элементов

Включи:
- Процент выполнения
- Streak (серии дней)
- Празднования (каждые 10 пунктов, каждую неделю)
- Визуальные индикаторы (✅ ⏸ ⏳)

### Шаг 5: Создание шаблонов

Создай шаблоны, которые можно использовать:
- Еженедельный обзор
- Месячный обзор
- Квартальный обзор (для OKR)
